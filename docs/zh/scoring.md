# 分数说明

## 基准定义

SigmaZ 的评分系统以 1994 年的主流高端配置 **Intel 486 DX2-66** 为基准（估算值）。该配置下的标准得分为 **100 分**。

## 计算公式

单项分数的计算采用归一化处理：

```
单项得分 = ( 测试结果数值 / 486基准参考值 ) * 100
```

例如，如果您的系统的整数运算速度是 486 DX2-66 的 50 倍，则该项得分为 5000 分。

## 参考数值表

| 测试项目 | 单位 | 486 参考值 (100分) | 权重 |
| :--- | :--- | :--- | :--- |
| INT (整数) | OPS (每秒操作数) | 2000.0 | 20% |
| FLOAT (浮点) | Iter/ms (每毫秒迭代) | 25.0 | 20% |
| MEM OPS (内存操作) | MB/s (兆字节每秒) | 20.0 | 10% |
| CRYPTO (加密) | Bytes/s | 500 KB/s | 15% |
| COMPRESS (压缩) | Bytes/s | 300 KB/s | 15% |
| MATRIX (矩阵) | Matrices/s | 0.3815 | 20% |

## 总分计算

为了避免单项性能极其突出（或极弱）导致总分失真，总分采用加权几何平均数计算：    

```
Total = exp(
  0.20 * ln(Int) +
  0.20 * ln(Float) +
  0.10 * ln(Mem) +
  0.15 * ln(Crypto) +
  0.15 * ln(Compress) +
  0.20 * ln(Matrix)
)
```

## 超时结算

针对早期的老旧硬件（如 386/486），SigmaZ 引入了超时结算机制：

*   整数、浮点、加密、压缩、矩阵测试在超过 **20秒** 时会自动停止。
*   内存测试采用固定时长窗口（约 **3秒**）以测量稳定带宽。
*   程序会根据已完成的计算量比例结算分数。
