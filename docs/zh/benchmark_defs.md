# 测试项目

SigmaZ 包含六项核心基准测试，分别针对 CPU 的不同计算单元和内存子系统进行评估。

### 1. Integer (整数运算)

**测试内容：** 基于 Machin 类公式计算圆周率 (Pi) 至小数点后 2000 位。

**算法公式：**
```
pi/4 = 4 * arctan(1/5) - arctan(1/239)
```

**技术指标：** 
-   **处理器单元**：算术逻辑单元 (ALU)，整数流水线。
-   **多核支持**：Win32/Win64 版本支持多线程并发计算。
-   **实现细节**：使用自定义的大数运算库（基于 `long` 类型数组）以确保在 16 位和 32/64 位平台上结果一致。

### 2. Float (浮点运算)

**测试内容：** 使用双精度浮点数生成 Mandelbrot 分形图像数据。

**算法公式：**
```
z(n+1) = z(n)^2 + c
```

**技术指标：** 
-   **处理器单元**：浮点运算单元 (FPU)。
-   **硬件检测**：对于早期系统，它能有效检测协处理器 (如 80387) 的性能。
-   **计算规模**：标准工作集为 16KB。现代处理器通过数万次循环验证吞吐量，老旧处理器执行单次验证。

### 3. Memory (内存吞吐)

**测试内容：** 通过 Stream Copy 算法对大块连续内存进行流式读写操作。

**技术指标：** 
-   **测试目标**：RAM 内存带宽，总线速度。

### 4. Crypto (加密解密)

**测试内容：** 使用标准查表法执行 CRC32 哈希计算。

**算法公式：**
```
CRC(x) = M(x) * x^32 mod G(x)
```

**技术指标：** 
-   **测试目标**：位运算（异或、移位）及随机内存访问（查表）。
-   **负载特征**：不仅考验 ALU，也考验 CPU 处理逻辑运算和缓存查找的效率。

### 5. Compress (数据压缩)

**测试内容：** 运行简化版的 LZ77 (LZ77-Simp) 无损压缩算法。

**算法机制：** 在历史窗口（4KB）中搜索当前字符串的最长匹配。

**技术指标：** 
-   **测试目标**：分支预测，复杂的控制流，非顺序内存访问。
-   **负载特征**：该测试对 CPU 的 L1/L2 缓存命中率非常敏感，能模拟实际的文件压缩工作负载。

### 6. Matrix (矩阵计算)

**测试内容：** 执行 O(N^3) 复杂度的密集矩阵乘法 C_(ij) = sum A_(ik) B_(kj)。

**计算规模**：
-   Win16: 64x64 矩阵
-   Win32/64: 64x64 矩阵
-   所有平台均计算 64x64 矩阵。Win32/64 通过高频重复运算测试 ALU 吞吐峰值。

**技术指标：** 
-   **测试目标**：CPU 缓存层级 (L1/L2/L3)，浮点吞吐量。
-   **优化潜力**：在现代处理器上，该测试能体现 SIMD 指令集 (如 SSE/AVX) 的自动向量化优化效果（如果编译器和操作系统支持）。
