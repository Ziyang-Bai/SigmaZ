# 疑难解答

**程序在 Windows 3.1 下崩溃或退出**

这通常是由于系统资源（GDI 堆或 User 堆）耗尽，或者是内存不足导致的。请尝试关闭其他正在运行的程序。如果问题依旧，你可以尝试增加分配给 Windows 3.1 的内存，或者参考下面的堆栈溢出解决方案。

**点击开始后界面无反应 (假死)**

这是正常现象。在单核且没有抢占式多任务处理的老旧操作系统上，密集的计算线程会暂时阻塞消息循环。程序内部已包含心跳机制，计算完成后界面会自动刷新，您放心。

**分数显示为 0 或极大**

如果某项测试分数显示为 0 或极大（如内存项目在Windows98上使用16位时分值高达10239950），可能是因为运行时间过短（小于计时器精度）导致无法计算有效分数。这通常发生在用极其现代的硬件（VMware虚拟机）运行为古董机设计的 `sigma16.exe` 时。建议改用 `sigma32.exe` 或 `sigma64.exe`。

**堆栈溢出 (Stack Overflow) 错误**

在部分 Win16 环境下，默认的栈空间可能不足。如遇此错误，由于 Win16 的内存模型限制，无法通过简单的编译选项调整栈大小。如果可能建议使用 `sigma32.exe` 或 `sigma64.exe`，如果您在 Windows 3.1 上运行，请尽量分配更多内存，如果仍然无法解决请尝试使用 `DRWATSON.EXE` 抓取寄存器快照汇报到 GitHub issues 以供分析。
